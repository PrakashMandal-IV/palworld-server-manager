# Palworld Server Manager

一款适用于 Windows 和 Linux 的桌面应用，让你无需命令行、无需手动编辑配置文件，就能轻松运行一个或多个 **Palworld 专用服务器**。安装后，将应用指向你的服务器（新建或已有的），即可在简洁的界面中管理一切。

## 功能特性

- **新建服务器**：通过 SteamCMD 新建服务器，或 **接管已有的** Palworld 专用服务器安装（保留你的世界、设置与管理员密码）。
- **启动 / 停止 / 重启 / 更新**：一键操作每个世界。崩溃守护程序可在服务器宕机时自动重启；可选的 **自动更新** 让世界始终保持在最新的 Palworld 版本——并在重启生效前于游戏内向玩家发出警告。
- **完整设置编辑器** —— `PalWorldSettings.ini` 中的每一项选项（100+ 项设置）被分组为易读的板块，支持搜索、单字段重置，以及经过社区验证的预设。只写入你修改过的设置，因此不会干扰其它内容。
- **玩家** —— 查看在线玩家；通过官方 REST API 踢出 / 封禁 / 解封。
- **实时地图** —— 在地图上实时标绘在线玩家。
- **控制台** —— 实时服务器日志流。
- **备份** —— 创建、恢复并定时备份世界。
- **计划任务** —— 按固定间隔或指定时间自动重启 / 备份。
- **模组** —— 导入并开关服务器模组。
- **聊天与广播** —— 实时查看游戏内聊天，并向玩家发送公告。
- **Discord 通知** —— 将服务器事件（启动、停止、重启、崩溃、备份、更新）推送到 Discord，并转发游戏内聊天。每个世界可添加多个 Webhook **频道**，并将每个事件路由到你指定的频道。
- **Discord 机器人** —— 用你自己的机器人通过斜杠命令管理世界：`/start`、`/stop`、`/restart`、`/broadcast`、`/backup`、`/status`、`/kick` 以及 `/player-location`（发布实时地图，标注每个在线玩家的点位与名称）。你可以指定哪些人可以使用每条命令；机器人只响应自己的命令。
- **多语言** —— 应用支持 English、Español、日本語、中文，可在设置中切换并即时生效。也可一键从 GitHub 安装更多社区翻译包，或自带翻译包——无需重启。
- **自定义** 每个世界的头像、横幅与主题色。
- **导出 / 导入** 设置及完整世界配置为 zip 文件，便于分享或在不同机器间迁移。
- **多世界并存**，每个世界拥有独立端口（自动分配以避免冲突）。
- **安全删除** —— 删除服务器文件、模组或存档时，会将其移入回收站（Windows）或废纸篓（Linux），因此误操作也可恢复。

## 下载

从 [**Releases**](https://github.com/PrakashMandal-IV/palworld-server-manager/releases/latest) 页面获取最新安装包：

- **Windows（安装程序）：** `Palworld Server Manager Setup <version>.exe`
- **Windows（便携版，免安装）：** `PalworldServerManager-<version>-portable.exe` —— 免安装运行，所有数据保存在 `.exe` 同级的 `PSM-Data` 文件夹中，因此可用 U 盘随身携带（连同你的世界）或在多台电脑间使用。
- **Linux：** `Palworld Server Manager-<version>.AppImage`

> Windows 构建尚未进行代码签名，因此 SmartScreen 可能会显示"未识别的应用"警告。点击 **更多信息 → 仍要运行** 继续。

## 快速开始

1. 使用提供的安装程序（Windows）或 AppImage（Linux）**安装** 应用。
2. 首次启动后你会看到 **你的世界**。点击 **新建世界** 创建一个，或使用 **使用已有的** 接管你已有的服务器（例如位于 `Steam\steamapps\common\PalServer` 下的）。
3. 世界出现在列表中后，点击 **启动**。首次启动服务器初始化时可能需要片刻。
4. 打开一个世界，使用各个标签页 —— 概览、玩家、广播、聊天、控制台、设置、备份、计划、模组、Discord、管理 —— 进行管理。

## 连接到你的服务器

打开一个世界，查看"概览"标签页上的 **连接** 框。在同一台电脑上，玩家通过以下地址加入：

```
127.0.0.1:<游戏端口>     （例如 127.0.0.1:8211）
```

在 Palworld 中：**加入多人游戏 → 通过 IP 连接**，然后粘贴该地址。

### 让好友通过互联网加入

默认情况下，你的服务器只能在本地网络中访问。要对外开放，你可以在路由器上设置端口转发，或使用免费的隧道服务。应用在 **信息** 板块中内置了 **playit.gg** 的分步指南（一种无需改动路由器的免费方案）。这是建议项，并非必需。

---

## 专用服务器 vs 社区服务器

**社区服务器** 与专用服务器相同，只是它还会出现在 Palworld 游戏内的公共服务器浏览器中，任何人都能发现并加入。它通过一个启动参数切换。**私有 / 专用** 服务器仅可通过 IP 加入。无论哪种方式，应用的管理方式一致 —— 在"管理"标签页中按世界切换。

---

## 关于设置的一点说明

Palworld 仅在服务器**启动时**应用服务器设置，因此修改设置后必须**重启**世界才能生效。应用只写入最小配置（仅你修改的部分），与 Palworld 自身存储设置的方式一致 —— 因此你已有的值及任何游戏内选择都会被保留。

端口、REST API 与管理员密码由应用自动管理，不会显示在设置编辑器中，因此不会因误操作而损坏。

---

## 数据与存储

应用将其注册表（你的世界列表及其元数据）存储在用户数据文件夹中：

- **Windows（安装程序）：** `%APPDATA%\palworld-server-manager\`
- **Windows（便携版）：** 便携 `.exe` 同级的 `PSM-Data` 文件夹
- **Linux：** `~/.config/palworld-server-manager/`

你实际的 Palworld 世界、存档与设置仍留在各自服务器的安装文件夹中 —— 应用从不移动它们。

---

## 运行要求

- Windows 10/11（64 位）或较新的 64 位 Linux 发行版。
- 为 Palworld 专用服务器及其存档预留足够的磁盘空间。
- 新建服务器时：需要联网（SteamCMD 会下载服务器）。

## 从源码构建

需要 Node.js 22.5+。

```bash
npm install
npm run dist:win      # Windows 安装程序 + 便携 .exe -> release/
npm run dist:linux    # Linux AppImage                -> release/
npm run pack          # 解包构建，用于测试            -> release/
```

在 Windows 上，首次打包构建请从 **以管理员身份** 打开的终端（或已启用开发人员模式）运行，以便 electron-builder 能够解包其工具链。

---

## 技术架构

Electron 外壳包裹一个自包含的 Next.js 服务器（App Router）。数据通过纯 WASM 后端存储在 SQLite 中，因此应用无需任何原生模块或数据库安装。所有 Palworld 管理均使用官方 REST API；已弃用的 RCON 协议默认关闭，仅可选开启。
